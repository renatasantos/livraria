Primeiros Passos

Nesta etapa queremos saber como você pensa, identifica cenários de problemas, especifica seus testes e criação de automação em tecnologias e metodologias que já lhe são familiares. Para isso, estamos enviando um pequeno desafio. Não se preocupe, o desafio é relativamente simples, mas importante para conhecermos suas habilidades.

Detalhes sobre o desafio:

Considere que você foi designado para uma equipe responsável pelo desenvolvimento de uma aplicação web de uma biblioteca. Inicialmente, sua equipe produziu um protótipo, uma versão minimamente viável e funcional do produto para validação por pessoas-chave do projeto.

Requisitos do sistema

Cadastro de novos usuários (CRUD):

  • Função: Cadastrar novos usuários na aplicação
  • Campos: nome do usuário, CPF, data de nascimento
  • Todos os campos são obrigatórios
  • Temos opções de Salvar, mas o botão só fica habilitado depois de todos os campos preenchidos
  • Na engrenagem podemos adicionar ou remover colunas da tabela
  • O campo de pesquisa busca por qualquer dado na tabela
  • Nos 3 pontinhos é possível editar ou excluir os dados da tabela

Cadastro de Autores

  • Função: Cadastrar novos autores na aplicação
  • Campos: nome do autor, nacionalidade
  • Todos os campos são obrigatórios
  • Temos opções de Salvar, mas o botão só fica habilitado depois de todos os campos preenchidos
  • Na engrenagem podemos adicionar ou remover colunas da tabela
  • O campo de pesquisa busca por qualquer dado na tabela
  • Nos 3 pontinhos é possível editar ou excluir os dados da tabela

Cadastro de Livros

  • Função: Cadastrar novos livros para a biblioteca
  • Campos: data da publicação do livro, data de inclusão, se ele está disponível para empréstimo, título do livro, autor (este campo está vinculado ao cadastro de autores, portanto só aparecerão autores caso existam no cadastro de autores), editora e observações
  • Todos os campos são obrigatórios e possuem uma validação para preenchimento deles
  • Na engrenagem podemos adicionar ou remover colunas da tabela
  • O campo de pesquisa busca por qualquer dado na tabela
  • Nos 3 pontinhos é possível editar ou excluir os dados da tabela

Atividades:

1 - Construir Casos de Teste:

Baseado nos requisitos do sistema, crie especificações de teste (casos de teste) para garantir a qualidade do software que está sendo construído. Caso conheça alguma metodologia para aplicação de testes, como por exemplo, Behavior Driven Design (BDD), você poderá aplicá-las se julgar necessário. Você terá a oportunidade de justificar suas escolhas.

2 - Construir Teste Automatizado:

Utilizando a especificação de testes criada, crie e execute alguns cenários de testes automatizados que validem o funcionamento do sistema, utilizando a aplicação.

Atenção, o sistema disponibilizado não contém todos os requisitos do software, por se tratar de um protótipo. Contudo, os seus testes automatizados podem ser construídos para validar comportamentos e regras que serão construídos posteriormente.

3 - Melhorias para o sistema:

Baseado nos requisitos do sistema, descreva melhorias e/ou regras de negócio que podem ser criadas para o projeto.

Envio do desafio:

  1. Crie um arquivo readme.doc com as soluções propostas para o desafio, como também com explicações de como rodar a automação de testes. Sinta-se à vontade também para expor suas ideias caso ache necessário, como por exemplo, o emprego de alguma metodologia de teste.
  2. Para fazer a entrega do desafio, compacte todo o conteúdo gerado (exceto arquivos compilados), incluindo o documento readme.doc, em um arquivo ZIP ou RAR e envie para os e-mails de contato. Podemos receber também via git, basta enviar o link.

Iremos avaliar suas habilidades em entender e propor cenários de testes, operar alguma ferramenta de automação de testes e resolução de problemas. Caso não consiga entregar o desafio completo, não deixe de nos enviar, iremos considerar os pontos em que conseguiu avançar.
